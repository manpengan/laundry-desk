# 证书信任演练（WSS 127.0.0.1）

## 目的

决定本地通道最终形态：

1. **方案 A**：`wss://127.0.0.1` + 安装器写入本机信任的本地 CA / 钉扎证书  
2. **方案 B**：`ws://127.0.0.1`（明文传输）+ **消息层加密/签名**（票据已有 server 签名）  
3. **方案 C**：仅 `app://` 壳内连 Edge（浏览器版另议）

架构默认写的是 WSS；本演练用数据推翻或确认。

## 步骤

1. `npm install && npm run cert`  
2. 确认 `channel/certs/meta.json` 的 `method=openssl`（不是 placeholder）  
3. `npm run wss`  
4. Chrome 打开 `https://127.0.0.1:17443/client`  
5. 记录是否出现「您的连接不是私密连接」  
6. 分别测试：
   - 用户点「继续前往」（临时例外）后 WSS 是否连通  
   - **不**添加例外时，是否完全失败  
   - 将证书导入「受信任的根证书颁发机构」后是否无警告  
7. Edge 浏览器重复 4–6  
8. 用 `file://` 打开 `browser-client.html` 连 `wss://127.0.0.1:17443`（Origin=`null`）

## 判定建议（供 Claude 验收）

- 若「导入根证书」在无管理员权限 Windows 上失败率高 → **倾向方案 B 或 C**  
- 若安装器可静默写入企业信任/用户信任存储 → 方案 A 仍可行  
- 无论 A/B，Origin 白名单 + nonce/seq/exp **都必须保留**（ADR-01）
