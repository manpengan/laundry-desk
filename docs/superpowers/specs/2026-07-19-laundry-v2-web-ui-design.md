## 1. 设计原则（按优先级排序）

1. **柜台速度高于一切**：高频路径（开单/取衣）以"秒"计成本。键盘与扫码优先——F1–F11 切服务大类（继承顺科肌肉记忆）、助记码检索、扫码即定位、`Enter` 逐级推进、数字小键盘友好。鼠标是兜底不是主路径。
2. **大目标 + 高对比**：柜台屏幕距离远、环境亮、可能戴手套。主操作按钮 ≥ 44px 高，票号/金额用大号等宽数字，状态用色块徽章不用纯文字。
3. **可回退**：任何多步流程可以后退修改而不清空（Curbside 被骂"一条道走到黑"的反面）。开单明细可随时点开重编辑；结算前一切可改。
4. **危险操作三件套**：独立权限位 + 必填原因 + 二次确认（对话框内复述影响面："将撤销票单 xxx，共 N 件、已收 X 元"）。绝不出现顺科管理工具那种 15 个等宽危险按钮并排的布局。
5. **状态诚实**：连接状态（在线/离线/N 笔待同步）、打印任务状态（排队/已出票/失败重试）常驻可见；失败绝不静默。
6. **AI 是助手不是主角**：AI 入口贴场景（按钮/chips/抽屉），不抢核心工作流；AI 产出永远可编辑、可拒绝；写操作走确认卡。
7. **中文优先**：术语对齐行业（收衣/取衣/水洗唛/挂点/交班），文案短句直述；i18n 结构预留但 v2 不做多语言。
8. **深浅色齐备**：柜台默认浅色（环境亮），老板端跟随系统；打印预览恒浅色。

同一 SPA，登录后按 `role × store_features` 渲染 IA；小程序独立应用。


## 3. 布局系统

- 栅格：12 列，间距 token 化（4/8/12/16/24/32）。
- 断点：`sm 640 / md 768 / lg 1024 / xl 1280 / 2xl 1536`。柜台三栏布局要求 ≥1280；1024–1280 折叠为两栏（右栏抽屉化）；<1024 柜台功能降级为"移动收银"简版（应急场景）。
- 桌面导航：左侧 64px 图标栏（可展开 208px），顶部为门店/员工/连接状态条；移动端底部 tab。
- 层级：页面 → 卡片（液态玻璃面板）→ 行内元素；模态尽量少，重编辑用**右侧抽屉**（保持列表上下文可见——反顺科的全弹窗交互）。

### 4.2 开单页（系统最高频页面）

三区布局，全程键盘可完成：

```text
┌── 左：品类选择 (span 5) ──┬── 中：本单明细 (span 4) ──┬── 右：客户与结算 (span 3) ──┐
│ 服务页签: 干洗 手洗 普洗…  │ 明细表(每行=一件):        │ [手机号/扫码/姓名 查找]      │

│ │ ¥18 ││¥25 ││ ¥35 │      │ 备注|改价(权限)|拍照       │ 原价 ¥53                    │
│ └────┘└────┘└────┘       │                          │ 折扣 -¥5 (会员卡自动/手输†)  │
│ 品类分类子页签:上装/下装/…  │ [暂存挂单] (有明细才出现)  │ 附加+¥3 加急+¥10 运费+¥0    │
│ [自定义品类…]             │                          │ 应收 ¥61  大号数字           │
│                          │                          │ 可取时间: 07-24 12:00 [改]  │
│                          │                          │ 付款方式(已启用的): 微信|现金 │
│                          │                          │ |余额*|记欠款|部分付款        │
│                          │                          │ ☑打取衣联 ☑打水洗唛 ☑回执   │
```

- **键盘流**：查客户(手机号+Enter) → F 键选服务 → 助记码打字选品类(Enter 加入) → 需要细化时 `Tab` 进抽屉 → `Ctrl+Enter` 结算。目标：熟手一单 ≤ 15 秒。
- **件级抽屉没有"数量"字段**（ADR-03：每件一条记录、独立条码）：数量在**加入品类那一步**输入（宫格确认弹层 qty 默认 1），一次生成 N 条件级记录，随后逐件编辑属性；明细表按件展示，同款可折叠显示 ×N。
- **AI 开单辅助**（`ai` 开启时）：抽屉内"📷 识别"按钮——拍照后品类/颜色/瑕疵以**候选 chips** 出现，点击采纳，绝不自动落库。
- 暂存挂单：确认弹层复述"暂存 N 件待续"，工作台出现"挂单中 ×N"角标可召回。
- 结算成功页：大字票号 + 取件码 + 打印状态行（小票✓ 水洗唛×3 出票中…失败可重打）。

- fulfillment 关闭时不显示挂点/工序徽章，件状态只有 已收/已取。

### 4.4 顾客页（散客 + 会员）

- 列表：搜索 + 标识筛选；行显示 姓名/打码手机号/累计消费/次数/上次；徽章：黑名单/欠款/会员。
- 详情抽屉：档案（多地址/车牌/证件按权限显示）、**客户级豁免开关**（不打小票/不打水洗唛/固定折扣）、历史订单时间线、欠款汇总与"生成催付信息"。
- `membership` 开启时：会员资产卡（余额=本金+赠额分列、次卡进度、积分、有效期）+ 操作组（充值/升级/挂失补卡/延期/合并/退卡——每个操作一个抽屉表单，高危项走三件套）。充值表单体现**金额三元组**：充值额/赠送额/实收额三字段联动，回执打印选项。

### 4.5 催取工作台（滞留即金钱）

- 过滤器：收衣日期区间 / 滞留天数分档(30/90/180+) / 未付款 / 状态；结果表含件数、金额、最后触达时间。
- 三档催取延续顺科：单个 / 按订单 / 按顾客合并；右侧预览面板显示将发送的文案（可点"AI 生成"按 分层语气 出稿，可编辑）。
- **通道降级**是显式 UI：微信/短信未开通时，主按钮变为"生成名单并复制号码"，导出 CSV，绝不假装能发。
- 发送=写操作 → 确认卡（人数/渠道/预估费用）→ 结果报表（成功/失败/费用）。

### 4.6 账目页

- 页签：今日 / 往日 / 月结 / 职员 / 交班*。
- 版式延续顺科被验证的三栏心智：左=流水式结账单（科目行）、中=渠道科目树（现金→洗衣/回收欠款…）、右=构成环图 + [打印] [导出] [详单]。
- 顶部**口径切换**：真实收入 / 业绩收入，旁附一句话解释（"业绩含刷卡消费、不含储值款"）——把顺科的口径说明产品化。
- 交班页（开启时）：上次/本次时间、应有现金 vs 实点现金（差额高亮）、预留备用金、[结账并打印]；历史结账单列表。

### 4.7 生产页*（fulfillment 开启时）

- 上挂：扫码连扫模式（三选一：仅显示/选中/直接上挂，继承顺科），格位选择器（分区×号位网格），语音播报开关，批量打不干胶。
- 店厂交接：建批次 → 扫码装包（实时清点 N/M）→ 出店；入厂/出厂/回店同构；差异（缺件/多件）红色阻断需处理。
- 工序看板：按 洗涤/熨烫/质检 列的看板视图，批量拖拽或扫码置状态。

### 4.9 AI 助手抽屉（全局）

- 入口：顶栏 ✨ 按钮或 `⌘K` 命令面板输入自然语言；从右侧滑出 400px 抽屉，**不遮挡工作区**。
- 消息流：用户气泡 / AI 流式文本（SSE 逐字）/ **工具执行行**（"⚙ 查询了 7 月营业额" 可展开参数与结果摘要）/ **确认卡**。
- 确认卡规范（写操作唯一通道）：

```text
┌ ⚠ 待确认: 发送催取通知 ─────────────┐
│ 对象: 32 位顾客(滞留>90天) [查看名单] │
│ 渠道: 短信   预估费用: ¥1.60         │
│ 文案预览: "您好!您有长期未取…" [✎改]   │
│ 操作者: 王店员 · 有效期 04:37 ⏳      │
│ [拒绝]              [确认执行] (主色) │
└────────────────────────────────────┘
```

**确认卡 = WYSIWYS 授权对象**（后端 `ai_pending_actions`，架构 §9.5），UI 契约：
- 卡面必须完整呈现被授权的内容：对象数量（可展开名单）、金额、渠道、完整文案——**用户看到什么就授权什么**，不允许"…等 N 项"式折叠关键参数。
- 卡绑定 `args_hash + 实体版本 + nonce + 有效期`：AI 换参数、涉及订单被并发修改、超时（默认 5 分钟，卡面倒计时）都会使卡**作废置灰**并要求重新发起；确认后参数不可变。
- 点"✎改"编辑文案 = 参数变更 → 旧卡作废、生成新卡（用户感知为"已更新预览"）。
- R4 工具的卡在"确认执行"后追加 **step-up 复核**：需**具备审批权限的另一身份**（店长/老板）现场 PIN 或扫码复核（同步，M2 起可用；绑定同一 canonical 参数与实体版本，原子单次消费），或转交异步审批中心（M5，卡面显示"已提交审批"状态）。操作者不能用自己的 PIN 复核自己。
- 确认与复核均只提交 nonce——**执行参数始终读取服务端冻结的 canonical 副本**，客户端不回传参数。

- 页脚恒显：`模型: DeepSeek V4 · 本次 ¥0.02`；每条 AI 建议附"复制/编辑/拒绝"。
- 报表页内嵌变体："AI 解读"按钮 → 在报表侧栏输出要点与异常提示（同一 agent，上下文带当前报表参数）。

### 4.10 老板端（H5）要点

- 首屏 4 张指标卡（今日营业额/取衣件数/新增欠款/滞留件）+ 7/30 日趋势迷你图；下拉刷新。
- AI 问答即首页第二屏（输入框常驻底部），预置问题 chips（"这周比上周如何？""哪些老客流失了？"）。
- 每周一推送"AI 经营周报"（模板消息→ H5 页）。
- 全部只读 + 营销发起；不承载柜台操作。

### 4.11 异步审批中心（M5，老板端为主 + 桌面后台同构；同步 step-up 复核自 M2 起在确认卡内完成，不依赖本页）

- 待审批队列：每条 = R4 动作卡（发起人/工具/影响面摘要/金额/过期时间），点开见完整参数（与确认卡同一渲染组件，保证"审的=执行的"）。
- 操作：[批准并执行] / [驳回(必填原因)]；批准即走命令总线执行并回执发起人；过期自动作废。
- 历史页签：全部审批记录（含自动化策略的创建审批），可按工具/人/时间筛。
- 通知：新审批到达 → 老板端角标 + 微信模板消息（通道开通时）。

### 4.12 小程序（P1）要点

- 衣物进度 = 件级时间线（已收衣→洗涤中→已上挂→已取回），有照片显示照片；对标"京东式透明"。
- 下单预约 = 选服务 + 期望时间段 + 地址（取送开启时）；未开启取送则为"到店登记"。
- 钱包页余额/次卡/券包；充值走微信支付（后置到支付资质就绪）。

## 6. 状态、反馈与文案规范

- 加载：卡片骨架屏；表格首载骨架、翻页保留旧数据（Query 的 keepPreviousData）。
- 空态：一句话 + 主行动按钮（"还没有价目，先添加品类"）；绝不空白。
- 错误：可重试错误给 [重试]；权限错误说明缺哪个权限位；表单错误行内定位。
- 打印失败：Toast + 顶栏红点 + 任务列表可逐条重打；连续失败提示检查硬件桥。
- 文案风格：动词开头、≤14 字按钮文案、金额一律 `¥` 前缀等宽；危险按钮用动词全称（"撤销票据"而非"删除"）。

## 7. 无障碍与工效

- 全键盘可达（焦点环可见）；触屏目标 ≥44px；对比度 WCAG AA；色觉安全（状态 = 颜色+图形双编码）；`prefers-reduced-motion` 尊重；扫码枪即键盘输入源，所有查找框兼容回车结尾的连续扫描。

## 9. 验收口径（UI 门禁附加项）

1. 开单页熟手键盘流实测 ≤ 15 秒/单（种子价目下脚本化度量）。
2. 断网演练：拔网线开单→打印→恢复→同步报告正确。
3. 深浅色双模式全页面走查无对比度断裂。
4. 危险操作全部命中三件套模板（代码审查 + E2E 断言）。
5. AI 写操作在**既无确认卡、又无生效自动化策略授权**时无法产生任何数据库变更（契约测试）；红线工具（退款/免单/余额/权限/密钥/备份恢复）在任何配置下不可自动执行。
6. 确认卡 WYSIWYS 断言：参数变更后旧卡必须作废并置灰（E2E）；过期卡不可执行。
7. 跨租户负向 E2E：A 店会话在任何 UI 入口（含 AI 抽屉与导出）拿不到 B 店数据。
