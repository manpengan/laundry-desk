## 2. M0 任务（spike）

### M0-3 三类打印机实机（`tools/spikes/m0-3-printers/`）

- 用宏发现役三台（manpengan 协助接机）：XP-58 小票（ESC/POS）、DASCOM DL-206 水洗唛（含**切刀**指令）、Gprinter GP-3120 不干胶（TSPL）
- 渲染含全变量的样张：水洗唛 21 变量 / 不干胶 22 变量（清单见架构 §与顺科矩阵报告）
- **验收**：三台各出正确样张；切刀动作正确；变量渲染无误；每台的指令族与坑记录进 `docs/research/2026-07-19-v2-m0-findings.md`

### M0-4 Edge 本地通道 + 冷启动 + A/B 升级（`tools/spikes/m0-4-edge/`）

- Windows 实测：浏览器 ↔ `wss://127.0.0.1` 的证书信任方案、Chrome Local Network Access 权限行为、防火墙提示——结论决定本地通道最终形态（WSS 证书 vs localhost WS + 消息层加密）
- `app://` 自定义协议加载**内置签名 SPA**：断电重启 + 断网冷启动进入本地页面
- A/B 双槽升级演练：备槽安装 → 健康检查失败自动回原槽；本地 SQLCipher 库快照与恢复
- **验收**：断网重启可进本地工作台；通道方案有结论与数据；回滚判定按支持矩阵语义执行；全程记录进 findings

## 3. M1 任务（生产代码）

### D 包 · `apps/edge-agent`（Edge v0）

- [ ] D1 Electron 壳：`app://` 内置签名 SPA、断网冷启动、**安全基线硬性达标**——`nodeIntegration:false`、`contextIsolation:true`、`sandbox:true`、`webSecurity:true`、最小 preload（白名单 IPC + sender 校验）、`setWindowOpenHandler`+`will-navigate` 禁任意导航/新窗口/外链、权限请求默认拒绝
