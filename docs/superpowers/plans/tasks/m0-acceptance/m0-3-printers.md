# M0-3 验收单：三类打印机实机

> 主责：**Grok**　验收：Claude　产出：`tools/spikes/m0-3-printers/`
> 依据：实施计划 §1.1、任务书 grok §2、架构 §10（签名模板本地渲染）、顺科矩阵报告 §3.1（现役机型与变量清单）
> 公共规则见 [README](README.md)。**本项是唯一含硬件的 spike，坑必须现在踩掉。**

## 1. 目标

用宏发现役三台实机验证三族指令流可被我们自己的代码驱动、模板全变量渲染无误，产出后续 D4（Edge 打印执行）与 M2 打印链路不需重新试错的实测记录。

## 2. 步骤

1. 接机（manpengan 协助）：`XP-58` 小票（58mm、USB、ESC/POS）、`DASCOM DL-206` 水洗唛（**含切刀**）、`Gprinter GP-3120` 不干胶（TSPL）。
2. 每台先跑最小字节流：初始化 → 文本 → 条码 → （DL-206）切刀 → 出纸；记录连接方式（直接 USB / 系统驱动 / 端口）。
3. 全变量样张：变量清单以顺科矩阵报告 §3.1（IMG_2315/2316）为准——**注意报告"21/22 个"计数与逐项列举存在出入，须以实拍界面核清并落出权威清单表**（该表即 v2 模板变量集的实测底稿）。
4. 边界样张各至少一例：中文长文本、空值变量、特殊字符（`@`、换行）、最长字段截断行为。
5. 样张数据**一律虚构**（虚构号段、假名、假条码）。
6. 记录每台：指令族与方言、纸宽/分辨率、浓度/速度参数、坑（编码、切刀时序、驱动安装等）。

## 3. 通过标准（逐条判定）

- [ ] 三台各出 ≥1 张全变量正确样张（实物照片为证）。
- [ ] DL-206 切刀动作正确：切在张与张之间、不切到内容。
- [ ] 全部变量逐一渲染无误：提交"变量 → 输入值 → 样张呈现"对照表，逐项 ✓；中文无乱码。
- [ ] 空值变量行为明确（留空/整行跳过）并已记录。
- [ ] 样张条码可回读（现场有扫码枪用扫码枪；无则手机扫码 app 验证并注明方式）。
- [ ] 水洗唛/不干胶**权威变量清单表**产出（含与顺科报告计数出入的说明）。
- [ ] 指令族与坑记录完整到"D4 实现者不需再对着打印机试错"的程度。

## 4. 证据格式

- `tools/spikes/m0-3-printers/README.md`：接机方式、依赖、逐台复现步骤。
- `evidence/`：样张实拍照片（**先去 EXIF、数据全虚构**——仓库 PUBLIC，RFC 素材裁决）；每台发送的字节流 dump（`.bin` + 生成脚本）；变量对照表。
- findings `## M0-3` 小节：结论行 + 三台"指令族/方言/坑"表 + 权威变量清单表。

## 5. 不通过 / 需改设计

任一台无法用自研字节流驱动（只能走厂商闭源驱动打印）→ 报「需改设计」，Claude 评估对架构 §10"Edge 本地渲染 ESC/POS/TSPL 字节流"的影响并起草 ADR。单台硬件故障不算不通过，记录后与 manpengan 约替代机时间。
