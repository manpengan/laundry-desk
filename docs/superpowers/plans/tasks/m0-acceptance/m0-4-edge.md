# M0-4 验收单：Edge 本地通道 + 冷启动 + A/B 升级

> 主责：**Grok**　验收：Claude　产出：`tools/spikes/m0-4-edge/`
> 依据：ADR-01（后果条款：M0 必测三件）、ADR-08（升级状态机）、架构 §10/§13.3/§13.5、任务书 grok §2
> 公共规则见 [README](README.md)。

## 1. 目标

Windows 实机回答三个悬而未决的设计问题：**(a)** 浏览器 ↔ Edge 本地通道最终形态（`wss://127.0.0.1` 证书方案 vs `localhost` WS + 消息层加密——ADR-01 明言由本实测定夺）；**(b)** `app://` 内置签名 SPA 断网冷启动是否成立；**(c)** A/B 双槽升级的回滚判定能否按支持矩阵语义执行。

## 2. 步骤

1. 环境记录：Windows 10/11 实机版本、Chrome/Edge 浏览器版本、防火墙状态。
2. **通道实测矩阵**（两方案都测，不许只测一个）：
   - 方案 A `wss://127.0.0.1` + 自签/本地 CA：证书生成与信任安装步骤数、Chrome 与 Edge 各自行为、证书过期/轮换的运维成本估计；
   - 方案 B `ws://localhost` + 消息层加密：HTTPS 页面连 `ws://localhost` 的混合内容拦截行为、Chrome **Local Network Access 权限**提示的触发条件/文案/拒绝后果；
   - 两方案下防火墙首次运行提示行为（是否触发、默认放行范围）；
   - 每方案记录：用户可见摩擦（弹窗数/步骤数）、失败模式、可自动化程度 → 给出**推荐结论与理由**。
3. **冷启动**：最小 Electron 壳注册 `app://` 协议加载安装包内置 SPA 静态资产 → 拔网线 + 强制断电重启 → 应能进入本地页面；DevTools offline 面板核对零网络依赖；记录冷启动耗时。
4. **A/B 双槽**：双版本目录 + 槽指针 + 健康检查脚本演练三条路径——升级成功切换；健康检查失败自动回原槽；SQLCipher 本地库快照 → 模拟损坏 → 恢复。
5. **回滚判定语义**：构造"旧版本不能读当前本地 schema"场景，断言判定走**恢复模式（只打印 + 只读）**而非盲目降级（ADR-08 第 8 条）。

## 3. 通过标准（逐条判定）

- [ ] 通道结论明确且有数据：两方案实测对比表 + 推荐方案 + 理由（此结论是 M1 A4 桥协议冻结的直接输入）。
- [ ] Chrome LNA 权限行为有第一手记录（触发条件、提示文案截图、拒绝后行为）。
- [ ] 断网 + 断电重启后进入本地工作台页面（录屏为证）；`app://` 加载零网络请求。
- [ ] 备槽健康检查失败 → 自动回原槽且原槽可用（演练日志）。
- [ ] 快照恢复：升级前快照 → 损坏 → 恢复后数据完整（校验查询一致）。
- [ ] "旧版读不了当前 schema"场景正确进入恢复模式判定分支，**未发生盲目降级**。
- [ ] 全部演练可复现（脚本或逐步 README）。

## 4. 证据格式

- `tools/spikes/m0-4-edge/README.md`：环境表、逐项复现步骤。
- `evidence/`：通道对比表（Markdown）、LNA/证书/防火墙提示截图（去个人信息）、冷启动录屏或逐帧截图、A/B 演练日志、快照校验输出。
- findings `## M0-4` 小节：结论行 + 通道推荐结论 + 三块演练结果摘要 + 坑。

## 5. 不通过 / 需改设计

两通道方案在实测中都存在不可接受摩擦（如企业无法静默装证书且 LNA 默认拒绝）→ 报「需改设计」，Claude 起草通道形态 ADR。冷启动或回滚判定不成立同样报「需改设计」——这两条直接动摇 ADR-01 §8 / ADR-08 的可行性，不允许带病进 M1。
