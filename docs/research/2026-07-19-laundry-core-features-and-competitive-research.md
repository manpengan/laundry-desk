> 日期：2026-07-19

## 1. 本店核心功能内核（已验证）

### 1.1 六个核心业务对象

洗衣店系统的一切功能都挂在这六个对象上，这是数据模型的根：

| 对象 | 内容 | 顺科的教训 |
|---|---|---|
| **顾客/会员** | 散客档案（姓名/电话/性别/车牌/地址/证件/标识）+ 会员资产（余额/赠额/积分/鞋次卡/衣次卡/有效期/黑名单） | 散客与会员共用 6 列基础画像，会员 = 散客 + 资产列；散客侧反而挂物流字段（车牌/地址）与客户级豁免开关 |
| **票单** | 交易载体：票单号、付款方式、金额汇总、状态、可取时间、回执标记 | 票据号在开单瞬间即分配；撤销必须填原因 |
| **单件衣物** | 最小业务对象：服务/名称/颜色/品牌/备注(瑕疵)/附加/条码/附件/款式/分区/数量/原价/折价 | **一票多件、逐件建档**是整个系统的地基；件级条码贯穿 收衣→打唛→上挂→取衣→补打→撤销 全链 |
| **收款** | 多渠道（现金/储值/微信/支付宝/…13 种可开关）、部分付款、未付款、欠款回收、金额三元组（充值/赠送/实收分离） | 宏发实况：6 开 7 关；现金收入 100% 来自回收欠款 |
| **通知** | 短信/微信双通道、12 个流程触发节点、催取三档（单个/按订单/按顾客）、生日/到期提醒 | 通道是否开通与功能存在是两回事，必须可降级（生成名单人工触达） |
| **账目** | 日/月/职员/交班四种口径、渠道科目树、**业绩收入 ≠ 真实收入**（排除储值款、包含刷卡消费） | 储值制下这个口径区分是会计正确性的关键 |

外加一个横切面：**门店配置**（价目字典 × 11 服务二维定价、卡类规则、颜色/品牌/备注/附加字典、打印模板变量、85 项分组权限、流程开关）。顺科把几乎所有业务字面都做成了可配置字典——这是它能卖给不同门店的原因。

### 1.2 业务闭环与实际使用度

| 闭环 | 顺科设计 | 宏发实际 | 对产品的含义 |
|---|---|---|---|
| A 收衣开单 → 打印 → 收款 | 完整 | ✅ 天天在用 | 产品生死线，体验要做到「数秒开单」 |
| B 取衣交付（多路查找/整单/部分/补款） | 完整 | ✅ 天天在用 | 同上 |
| C 欠款回收 + 库存催取 | 完整 | ✅ 在用（126 单欠款 / 1067 件滞留） | 中小店真实痛点，催取要能降级为人工名单 |
| D 生产流转（送洗/上挂/挂点） | 完整 | ⛔ 全线 0 件，流程开关关闭 | **必须可开关**：有工厂/输送线的店开，夫妻店关 |
| E 会员储值经营 | 完整 | ⛔ 仅 1 名会员 | 单店可以不用，**产品不能没有**（见 §2） |
| F 交班结账 | 完整 | ⛔ 从未结过账 | 雇人店才需要，可开关 |
| G 微信线上（下单/支付/裂变） | 完整 | ⛔ 五项全未开通 | 增值模块，不进核心 |
| H 权限与审计 | 85 项权限 | 部分（单人操作） | 产品必备，粒度可参考顺科分组 |

### 1.3 顺科做对的、必须继承的设计细节

1. **一票多件、件级条码**——物理追踪链的起点。
2. **五段计价**（原价/折价/附加/加急/运费）+ 核价总额，金额段分立不混算。
3. **金额三元组**（充值金额/赠送金额/实收金额分离），赠额可"月月到账/清零"。
4. **可取时间自动计算**（收到后 N 天 M 时），逾期概念由此而来。
5. **暂存挂单**（柜台被打断是常态，有确认流程）。
6. **撤销必填原因** + 撤销金额回冲科目（撤销未付/撤销微信）。
7. **打印模板变量系统**（水洗唛 21 变量 / 不干胶 22 变量，字段/字号/坐标/份数/时机全可配）。
8. **客户级豁免开关**（不打印取衣单/不分配挂点/不打印水洗唛）。
9. **流程节点可开关**（收衣/送洗/上挂/取衣逐个开关）——同一套软件适配不同业态的关键。
10. **权限按业务动作细分**，高危操作（改价/免单/改余额/删客户/跳过密码）单独成组。
11. **业绩收入 ≠ 真实收入**的报表口径。
12. **助记码检索 + F1–F11 快捷键**——柜台速度来自键盘而非鼠标。

反面教材（不继承）：管理员工具 15 个高危按钮无逐项确认；备份无限堆积无恢复演练；所有配置塞进模态弹窗的交互；手机号在列表中全量明文。

## 2. 定位转变：从单店工具到多店产品

用户决策：laundry-desk 将统一架构升级后**提供给洗衣店行业使用**。这个转变直接翻转三个此前的优先级结论：

| 能力 | 单店视角（此前结论） | 产品视角（新结论） | 理由 |
|---|---|---|---|
| **会员储值** | 不做（宏发仅 1 名会员） | **P0 核心** | 储值 = 洗衣店现金流命脉，是店主买软件的第一动机；国内全部竞品都有，通用会员 SaaS（云上铺/纳客）甚至只靠它切市场 |
| **生产流转/店厂协同** | 不做（宏发 0 使用） | **可开关的标准模块** | 国内干洗本就是"门店收衣 + 工厂集中洗"结构；傲蓝/洗衣管家都以工厂版区分产品线；夫妻店关掉即可 |
| **微信生态** | P3（宏发未开通） | **主战场** | 顺科最新版增量全部押在微信端与连锁云，桌面端已停止进化；国际对标 QDC 的 WhatsApp-native 范式 |

不变的结论：**衣物级追踪链（单件明细→条码→标签打印→部分取衣）无论单店还是产品都是 P0**；硬件重投入（RFID/取衣柜/输送线）只做接口预留。

## 3. 国内竞品格局

### 3.1 市场分层

```text
老单机守存量        顺科(25.3.x 桌面停止进化)、天籁、洗衣掌柜
                    买断 1500–2200 元/套；增量全在微信公众号 + 连锁云("一天一元起")
─────────────────────────────────────────────────────
全家桶收割升级      傲蓝(门店/工厂/布草/收衣柜四线 + 洗晴新品牌)
                    洗衣管家(宣称 4 万门店；门店+小程序+员工/物流/工厂/柜机全矩阵)
                    定价不公开，销售驱动，靠 营销套餐/工厂版/硬件 分层加价
─────────────────────────────────────────────────────
低价横切            通用会员 SaaS(云上铺免费版/纳客 ~300 元/年)——会员储值强、衣物流程浅
                    小程序模板商(SaaS 698–5998 元/年)——用户端体验好、门店端浅
─────────────────────────────────────────────────────
平台教育用户        美团/抖音团购(引流+评分信任)、京东洗衣(按件透明价+三重赔付)、
                    e袋洗(按袋计费+社区取送)——把"进度可查、上门取送、透明定价"打成用户预期
```

**空档判断**：老单机与全家桶之间存在「**轻量现代化**」空档——把 微信通知/顾客小程序/老板手机端/营销工具箱/权限审计 做扎实、体验现代化、定价透明，即可对中小店形成压倒性体验优势。这正是 laundry-desk 的切入位。

### 3.2 国内竞争力功能 TOP（★入场券 ◆标配 ▲差异化）

| 级 | 功能 | 竞品覆盖 |
|---|---|---|
| ★ | 收衣核心流：数秒开单、拍照留证、三类打印、条码、挂位、无票多路取衣、部分取衣 | 全部 |
| ★ | 会员储值：多卡种/充值赠送/挂失补卡/积分/无卡消费/计次卡 | 全部 |
| ◆ | 微信自动通知：完工/催取/回执，短信兜底（106 通道+空号检测） | 顺科微信版/傲蓝/洗衣管家/洗衣掌柜 |
| ◆ | 顾客小程序/公众号：查进度/余额/记录、在线下单、在线充值 | 洗衣管家宣称 85% 商户选装 |
| ◆ | 连锁多店：云同步、跨店消费、总部报表 | 顺科连锁云/傲蓝/洗衣管家(75% 选多店版) |
| ◆ | 老板手机端：营业额/会员/订单远程看 | 洗衣管家商家版/云上铺/洗晴 |
| ▲ | 营销工具箱：券/满减/秒杀/拼团/抽奖/**分享裂变推荐奖励**/套餐卡 | 傲蓝营销套餐(单独收费)/洗衣管家/云上铺 |
| ▲ | 上门取送闭环：预约→员工端抢单+导航→上门开单→进度反馈 | 傲蓝/洗晴/洗衣管家物流版 |
| ▲ | 平台对接：美团/抖音/口碑团购核销，顺丰/京东快递下单 | 傲蓝专业版/洗晴/洗衣管家 |
| ▲ | 工厂版工序管理：分拣→洗涤→质检扫码溯源（洗衣管家宣称 18 工序 32 节点） | 傲蓝/洗衣管家/洗衣掌柜 |
| ▲ | AI 查衣（掉标衣物以图搜图定位订单，洗衣管家 2024-07，宣称误判率<0.1%） | 仅洗衣管家——全场唯一 AI 落地点 |
| ▲ | 智能收衣柜 / RFID 布草 | 傲蓝/洗衣管家（重资产，后置） |

### 3.3 值得注意的单点

- 顺科官网仍在卖「**同行软件数据破解导入**」作为迁移抓手——数据迁移工具是抢存量客户的武器，产品化时要做正向版本（合规导入器）。
- 顺科短信卖点里有「**空号停机检测**」——群发前清洗号码，成本敏感型门店在意。
- 「洗衣通」不是竞品（实为洗护社区+商城 App）；「澜海/衣道/智洗/洁到家」未查到成型产品。

## 4. 国际竞品格局

### 4.1 分层与资本动向

- **SMB 全能 SaaS**：CleanCloud（5000+ 商家，事实标准）、Turns（已并入机器支付公司 PayRange）、Geelus（定价最透明 $19–129/月）、EnLite POS
- **企业级干洗**：SPOT by Xplor、SMRT（RFID+路线最强）、Compassmax
- **自助洗衣房 OS**：Cents（2026-04 完成 $140M C 轮，行业最大单笔；已吞并 Starchup）
- **取送/WDF 专精**：Curbside Laundries（$260/月 flat）、WashClubTrak（订阅制+超量计费样板）
- **新兴市场**：QDC（印度，WhatsApp 原生 + **免费 100 单/月起按单计费**，靠此吃下下沉市场）

定价基准 $49–499/月/店，普遍另收 SMS/品牌 App/交易抽成。

### 4.2 国际竞争力功能 TOP（附中国适配判断）

| # | 功能 | 谁有 | 中国适配 |
|---|---|---|---|
| 1 | **上门取送三件套**：顾客预约+路线优化+司机 App（地理围栏/到达前短信/拍照签名） | 全部 10 家 | 强适配——小程序预约+骑手端；国内竞品最弱、国际最成熟的一块 |
| 2 | **顾客自助端**：品牌 App/网页下单、全程状态推送 | 全部 | 强适配且降维：微信小程序天然替代品牌 App（国际上是高价加购项） |
| 3 | **生命周期营销 Flow**：生日券、沉睡唤回（QDC 宣称 18–25%）、好评引导/差评拦截 | CleanCloud/Cents/SPOT/QDC/Geelus | 强适配：通道换微信模板消息+企微；评价引导换大众点评/抖音。国内无人做成产品化 Flow |
| 4 | **订阅制经常性收入**：包月洗衣+超量计费、绑卡自动扣款 | WashClubTrak/Cents/CleanCloud/Turns | 适配需本土化：中国已习惯储值，机会是"家庭袋洗月卡"+微信自动续费 |
| 5 | **衣物级追踪**：条码/热封标签/RFID + 拍照留证 | SPOT/SMRT/EnLite/QDC 等 | 强适配（已在 P0） |
| 6 | **第三方运力集成**（DoorDash/Uber 零成本配送） | CleanCloud/Turns/Curbside/Cents | 强适配：同构方案 = 美团跑腿/闪送/达达聚合 API |
| 7 | **AI 客服**：AI 语音接线员 + chatbot | CleanCloud（$110/月档已含）、Cents Assist | 改形态：国内做微信 AI 客服（报价/催取/洗护问答） |
| 8 | **按单量计费**：免费 100 单/月起步 | QDC、CleanCloud 按单量分档 | 强适配：破解小店对固定月费的抵触，获客利器 |
| 9 | **多门店/中央工厂路由** | CleanCloud Multi-Store Plant/QDC(1500 店加盟) | 适配（连锁期） |
| 10 | **IM 原生运营范式**（WhatsApp-native：bot 下单/支付链接/状态推送/评价邀请） | QDC 最彻底 | **最值得整体照搬**：1:1 映射微信生态，是中国系统的主场优势 |
| — | 洗衣机 IoT/传送带自动分拣 | Cents/CleanCloud/SMRT | 弱适配/工厂专属，不碰 |

### 4.3 评价里骂出来的坑（产品红线）

- 强依赖网络断网即瘫（CleanCloud）→ **必须有离线兜底**（Turns 有离线模式做卖点）
- 报表导不出/对不上账（SPOT/Cents/QDC 均被骂）→ 导出与对账口径是基本功
- 操作线性不可回退改错（Curbside）→ 单据编辑流要可逆
- 促销码 bug 锁死顾客（CleanCloud）→ 营销引擎要有兜底解锁
- 数据安全疑虑（Turns 用户："感觉厂商员工能拷我数据"）→ 数据归属与导出自由要写进承诺

## 5. 技术趋势结论（按落地门槛分档）

**软件即可（高杠杆区，全部值得做）**：
1. 拍照证据链（收衣拍照挂到件级记录——防纠纷第一武器，黑猫上洗衣平台近千条投诉多源于此）
2. 会员储值/次卡 + 自动提醒
3. 一衣一码条码 + 工序状态机（所有高级功能的地基；数据模型预留 `rfid_tag_id`）
4. 一键同城取送（微信即时配送接口 / 快递100 聚合 API，一次接入顺丰同城/闪送/达达；银豹 POS 已有先例）
5. 美团/抖音团购券核销对接（核销不入系统 = 会员流失）
6. LLM 应用：催取/营销文案生成、微信 AI 客服、AI 经营周报（CleanCloud 已验证付费意愿）
7. 店厂协同：批次交接单（出店/入厂/出厂/回店四节点扫码）+ 工序状态 + 双向对账
8. 先洗后付（微信支付分/芝麻先享，与取送场景天然契合）

**低成本硬件（行业标配，必须适配）**：标签/水洗唛/小票三类打印机、扫码枪、电子秤（WDF 按磅计价）、钱箱、语音播报。

**接口预留、暂不集成**：RFID（1.8–5 元/枚，仅高端洗护/工厂 ROI 成立）、智能取衣柜（对接丰巢式平台比自铺轻）、电动衣架/输送线（本质是"单号→位置"指令协议，CleanCloud×MetalProgetti 是成熟范式）。

**不做**：刷脸支付（热度回落、无洗衣案例）、自建运力、自研柜机/输送线固件、泰笛式平台梦（黑猫近千条投诉 + 人去楼空的前车之鉴）。

## 6. 产品功能蓝图（产品化视角的 P0–P3）

> 与 7-19 上午单店复核报告的差异：会员储值 从"不做"升为 P0；生产流转 从"不做"升为可开关模块；微信生态 从 P3 升为 P1 主战场。衣物追踪链维持 P0 不变。

### P0 产品地板（没有就卖不出去）

1. **衣物级数据模型**：一票多件、逐件 服务/名称/颜色/品牌/瑕疵/附件/数量/原价/折价 + 件级条码 + 件级状态机 + 照片绑定（预留 rfid_tag_id）
2. **收衣开单**：品类价目二维定价（服务×衣物）、助记码检索、快捷键、五段计价、暂存挂单、可取时间自动计算
3. **收款**：多支付方式可开关、部分付款、未付款/欠款、金额三元组
4. **三类打印**：小票 / 水洗唛 / 不干胶，模板变量系统 + 补打
5. **取衣**：多路查找（票号/条码/电话/姓名/取件码）、整单/部分取衣、取衣时补款
6. **会员储值**：多卡种（折扣/储值/计次/包月）、充值赠额（含月月到账）、挂失补卡、余额支付、积分
7. **微信通知 + 短信兜底**：完工/催取/回执模板消息；通道未开通时降级为人工名单
8. **员工权限 + 操作审计**：分组权限（参考顺科 7 组），高危操作（改价/免单/改余额/撤销/删除）单独授权 + 全量留痕
9. **账目**：日/月/职员三口径 + 渠道科目树 + 业绩/真实收入双口径；交班结账（可开关）
10. **数据备份 + 还原演练**、订单撤销（必填原因 + 回冲科目）
11. **老板端**（手机看营业额/订单/会员，成本低感知强）

### P1 标配竞争力（决定能不能打赢傲蓝/洗衣管家的轻量位）

1. 顾客小程序：查进度/余额/消费记录、在线充值、在线下单预约
2. 生命周期营销 Flow：生日券、沉睡唤回、储值到期提醒、好评引导
3. 库存催取自动化（三档催取 + 通道策略 + 逾期保管费规则）
4. 上门取送：聚合运力 API（闪送/顺丰同城/达达）+ 运费入单 + 轨迹回传
5. 团购核销：美团/抖音券核销入系统并沉淀客户
6. 优惠券/满减/套餐卡/分享裂变（单层推荐奖励即可）
7. 报表导出 + 财务对账口径（Excel/流水核对）
8. 生产流转模块（送洗/上挂/格位，可整体开关）

### P2 成长与差异化

1. 店厂协同：交接批次 + 工序状态上报 + 门店-工厂对账
2. 多门店连锁：云同步、跨店消费、总部价格体系与汇总报表
3. AI 三件：拍照识别品类作为开单候选（不自动定价）、AI 查衣（以图搜图找掉标件）、AI 客服/经营周报
4. 先洗后付（微信支付分）
5. 包月洗衣订阅（家庭袋洗月卡 + 自动续费）
6. 电动衣架/格位定位取衣接口
7. 电子秤集成（WDF 按磅）

### P3 接口预留 / 明确不做

- 预留：RFID、智能取衣柜（丰巢式对接）、输送线协议、开放 API
- 不做：刷脸支付、多级分销（合规风险）、自建运力、自研硬件

## 7. 对后续架构升级与 Web UI 的设计输入（预告）

架构设计文档需要回答的问题（由本调研直接导出）：

1. **形态**：Web-first + 桌面壳（打印/外设桥）——顺科证明桌面端已停止进化，但打印机/扫码枪/钱箱要求本地硬件桥；CleanCloud 被骂"断网即瘫"证明**离线兜底**是必答题。
2. **多租户**：单店单库起步，但账号/门店/权限模型一开始就按多店设计（避免傲蓝式"版本线分裂"）。
3. **模块开关**：流程节点、会员、交班、生产流转全部做成 per-store 开关（顺科流程开关 + 付款方式开关的推广）。
4. **状态机地基**：件级状态机 + 票单状态机分离，是催取/店厂协同/RFID 的共同地基。
5. **集成层**：微信（模板消息/小程序/支付）、聚合运力、团购核销、LLM——全部走适配器，不硬绑厂商。
6. **数据承诺**：本地+云双备份、可还原、可导出——对标 Turns 用户的数据安全疑虑，把"数据是店主的"做成卖点。

Web UI 设计方案需要覆盖的四个界面形态：
1. **柜台工作台**（大按钮+键盘/扫码优先+常驻首页，参考顺科"员工长期停留首页"的设计）
2. **老板手机端**（看数 + 远程发活动）
3. **顾客小程序**（进度/余额/下单/充值）
4. **工厂/员工端**（扫码交接、工序上报，PDA/手机）

## 8. 三句话总结

1. **核心内核已验证**：六对象（顾客/票单/单件衣物/收款/通知/账目）+ 件级追踪链 + 12 条顺科设计细节，这是产品的地基，架构设计从这里出发。
2. **市场空档明确**：老单机停止进化、全家桶价格不透明且交互陈旧，「轻量现代化 + 微信原生 + 定价透明（可学 QDC 免费 100 单/月起按单计费）」是切入位。
3. **优先级已翻转**：会员储值升 P0、微信生态升主战场、生产流转变可开关模块；高杠杆功能全在"软件即可"区间，硬件只做接口预留。
