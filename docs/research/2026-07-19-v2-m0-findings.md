## M0-5: 三 adapter × 模型工具调用兼容矩阵

_主责：Gemini_

### 结论：【通过】

我们设计并验证了统一的消息信封，成功抹平了 `anthropic` (官方 SDK)、`openai-compat` (通用 OpenAI 格式，支持国产 DeepSeek/DashScope 环境变量路由) 和 `gemini` (使用最新的原生 `@google/genai` 2.12.0 SDK) 的调用细节差异。

### 六维方言与接口差异比对矩阵

| 比对维度 | Anthropic (Claude) | OpenAI / Compat (GPT/DeepSeek/Qwen) | Gemini (GoogleGenAI SDK) |
| :--- | :--- | :--- | :--- |
| **1. System 消息** | 必须作为 API 顶层 `system` 属性传入，不可混入 messages 数组 | 混入 messages 数组中，作为 `role: 'system'` 传入 | config 属性下的 `systemInstruction` 字段传入 |
| **2. Assistant 角色** | `'assistant'` | `'assistant'` | `'model'` |
| **3. 多 Tool 并行支持** | 允许，单回复 content 中返回多个 `{ type: 'tool_use' }` | 允许，Choice Message 中返回包含多元素的 `tool_calls` 数组 | 允许，候选 Parts 数组中包含多个 `functionCall` 元素 |
| **4. 工具定义格式** | 顶层 `name`, `description`, `input_schema` | 包装于 `type: 'function'` 之下的 parameters 规范 | Native 结构，定义在 `functionDeclarations` 中 |
| **5. 流式 Tool Delta** | 监听 `content_block_delta`，流式累加 `input_json_delta` | 监听 `delta.tool_calls` 各索引 chunk 累加 `arguments` 字符串 | 原生 SDK 的 `generateContentStream` 处理，Delta 级触发 `functionCalls` |
| **6. Token 用量字段** | `usage.input_tokens` / `usage.output_tokens` | `usage.prompt_tokens` / `usage.completion_tokens` | `usageMetadata.promptTokenCount` / `usageMetadata.candidatesTokenCount` |
