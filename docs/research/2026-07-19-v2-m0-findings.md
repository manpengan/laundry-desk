## M0-3 三类打印机（Grok）

### 产出位置

`tools/spikes/m0-3-printers/`

- 指令流生成器：XP-58 ESC/POS、DL-206 水洗唛（含 `GS V` / `ESC i` 切刀变体）、GP-3120 TSPL  
- 虚构全变量样张：`fixtures/sample-order.json` → `npm run generate`  
- 发送：`send/send-raw.mjs`、`send/send-windows.ps1`  
- 逐台清单：`ops/CHECKLIST-*.md`  
- 离线单测：金额整数分、切刀字节、22 不干胶变量、TSPL 头

### 变量集（相对顺科矩阵）

| 类型 | 矩阵文案 | 本 spike 实现 | 说明 |
|---|---|---|---|
| 水洗唛 | 21 个 | **23** 个命名 token（矩阵括号列表全量） | 文案计数与列表不一致；现场以实际打印机可印字段为准，回传后冻结生产模板 schema |
| 不干胶 | 22 个 | **22**（含 已消毒 / 打开存放 / 付款方式含欠款语义） | 与矩阵一致 |

金额全程整数分，仅渲染为 `¥x.xx` 文本。

### 指令族假设（待实机确认）

| 机型 | 假设指令族 | 编码 | 风险 |
|---|---|---|---|
| XP-58 | ESC/POS | GBK | Windows 非 RAW 驱动会吞指令 |
| DASCOM DL-206 | ESC/POS + 切刀 | GBK | 切刀字节可能是 `GS V` 或 `ESC i`；若实机为其它方言 → **需改设计** |
| Gprinter GP-3120 | TSPL | GBK 文本字段 | `SIZE`/`GAP`/字库名需按耗材校准 |

|---|---|---|---|---|

### 设计影响

- 当前：**无阻塞代码合并**（spike 独立目录）。

## M0-4 Edge 本地通道 + 冷启动 + A/B（Grok）

### 产出位置

`tools/spikes/m0-4-edge/`

| 子项 | 内容 |
|---|---|
| 通道 | `channel/wss-server.mjs`（Origin 白名单 + nonce/seq/exp）、证书生成、浏览器探针页、证书/LNA/防火墙清单 |
| 冷启动 | `cold-start/electron-app`：`app://` 加载内置 SPA、sha256 完整性占位、Electron 安全基线九项 |
| A/B | `ab-upgrade/drill.mjs`：双槽、健康检查失败回原槽、快照、支持矩阵回滚、anti-rollback、恢复模式 |
| Windows 总册 | `ops/WINDOWS-DRILL-RUNBOOK.md` |

### 离线已验证（开发机单测）

- 消息防重放：同 nonce 第二次 → `replay`  
- 队列非空 → 拒绝安装  
- health fail → **不切换**主槽  
- health pass → 切换备槽  
- matrix 不兼容 → **RECOVERY_MODE**（只打印+只读），禁止盲目降级  
- 低于 `minSecureVersion` → 拒绝安装  

### Windows 现场结果（待填）

| Chrome LNA | 待测 |
| 防火墙 loopback | 待测 |
| 断网冷启动 app:// | 待测 |

### 设计影响（预判，待数据）

- 架构默认 WSS；若店员侧证书信任成本高，**备选**为 loopback WS + 能力票据/回执已有的消息层签名（ADR-01 已要求票据），或浏览器版放弃本地硬件通道。  
- 冷启动路径与「禁止 Electron 加载远程 URL」一致；M1 D1 应直接继承本 spike 的安全基线清单。  
- A/B 回滚语义与 draft3.1a 一致：**无条件自动回滚不存在**。
