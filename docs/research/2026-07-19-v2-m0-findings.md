## M0-3 三类打印机（Grok）

### 产出位置

`tools/spikes/m0-3-printers/`

- 指令流生成器：XP-58 ESC/POS、DL-206 水洗唛（`GS V` / `ESC i`）、GP-3120 TSPL  
- 虚构全变量样张 + **边界样张**（empty / long / special）  
- CODE128 **强制 `{B`/`{C` 码集前缀**；默认 BC+GS w 1；另出 b/bc × w1/w2 变体 bin  
- fullvars 不干胶 **SIZE 40×90**（条码不再压出 40×60 下缘）  
- TSPL TEXT **引号转义**（`"` → `＂`，换行展平）+ 单测  
- 发送：Node `send-raw.mjs` **首选**；`send-windows.ps1` 在 PS5.1 拒绝不可靠 `-Port`  
- 现场清单含去 EXIF / 仓库 PUBLIC 红线  
- **无打印机适配**：`npm run lab:offline` = test + generate + `verify`（协议/￥/无 0x3F）+ `mock-send`（假 spool）；见 `ops/NO-PRINTER-LAB.md`

### 变量集（相对顺科矩阵）

| 类型 | 矩阵文案 | 本 spike 实现 | 说明 |
|---|---|---|---|
| 水洗唛 | 21 个 | **23** 个命名 token（矩阵括号列表全量） | 文案计数与列表不一致；现场后冻结模板 schema |
| 不干胶 | 22 个 | **22**（含 已消毒 / 打开存放 / 付款方式含欠款语义） | 与矩阵一致 |

金额全程整数分，渲染为全角 **`￥x.xx`**（U+FFE5）。半角 `¥`（U+00A5）不在 GBK，iconv 会落成 `?`——已踩坑并加 0x3F 防回归单测。

### 指令族假设（待实机确认）

| 机型 | 假设指令族 | 编码 | 风险 |
|---|---|---|---|
| XP-58 | ESC/POS | GBK | 无码集前缀则条码整条不打；纯 B+w2 超 384dot |
| DASCOM DL-206 | ESC/POS + 切刀 | GBK | 切刀字节可能 GS V 或 ESC i；若非 ESC/POS → 改设计 |
| Gprinter GP-3120 | TSPL | GBK 文本字段 | 物理标签高度不足时 fullvars 仅作指令流回归 |

## M0-4 Edge 本地通道 + 冷启动 + A/B（Grok）

### 产出位置

`tools/spikes/m0-4-edge/`

| 子项 | 内容 |
|---|---|
| 通道 | WSS + Origin 白名单 + nonce/seq/exp；证书/LNA/防火墙清单 |
| LNA | **L2 必须公网 HTTPS 或非 loopback origin**；明确 loopback→loopback 无效 |
| 冷启动 | `app://` 壳 + 安全基线九项；证据改为 **录屏优先** |
| A/B | 双槽、health fail 回原槽、快照损坏恢复、**裸 rollback 真矩阵**、anti-rollback |
| Windows 总册 | `ops/WINDOWS-DRILL-RUNBOOK.md` |
| 无 Windows 适配 | `npm run lab:offline` + `ops/NO-WINDOWS-LAB.md` |

### 离线已验证（开发机单测 / 无 Windows）

- 消息防重放：同 nonce 第二次 → `replay`  
- 队列非空 → 拒绝安装  
- health fail → **不切换**主槽  
- health pass → 切换备槽  
- **裸 rollback**：2.0.0 矩阵允许 → 回 1.9.0；2.1.0+contract → RECOVERY_MODE  
- 快照 → 人为损坏 → restore → **sha256 一致**  
- 低于 `minSecureVersion` → 拒绝安装  

### 复核修订

| ID | 项 | 状态 |
|---|---|---|
| P6 | 快照损坏→恢复→hash | 已修（runbook + 单测） |
| P7 | 裸 rollback 真矩阵路径 | 已修（`ab:rollback-matrix` + 单测） |
| P8 | LNA L2 公网源 | 已修（清单重写） |
| P9 | 冷启动证据录屏 | 已修 |
| P10 | engines ≥22.6 | 已修 |

### Windows 现场结果

| 项 | 结论 |
|---|---|
| 127.0.0.1 WSS 证书信任 | **挂起：无 Windows**（macOS 可做 L1 对照，不定案） |
| 推荐通道形态（A/B/C） | **挂起**（需 L2 + 证书 UX 数据） |
| Chrome LNA（L2 公网 origin） | 可在任意桌面 Chrome 补；**非** loopback→loopback |
| 防火墙 loopback | **挂起：无 Windows** |
| 断网冷启动 app://（录屏） | macOS Electron 可选；**OS 断电挂起** |
| OS 断电冷启动 | **挂起：无 Windows** |

离线替代证据：`npm run lab:offline`（单测 + 证书 + A/B 真矩阵回滚 + 快照 sha256）。

### 设计影响（预判，待数据）

- 架构默认 WSS；若证书信任成本高，备选 loopback WS + 消息层签名，或浏览器版放弃本地硬件通道。  
- 冷启动路径与「禁止 Electron 加载远程 URL」一致。  
- A/B：**无条件自动回滚不存在**；矩阵不兼容 → 恢复模式。
