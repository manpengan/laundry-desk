## 一、结论摘要

从截图看，这套软件不是单纯的“收衣开票工具”，而是一套面向实体洗衣门店的桌面经营管理系统。它覆盖了以下完整业务链：

1. 顾客识别与建档；
2. 收衣开单、衣物描述、定价与结算；
3. 送洗、生产完工、上挂、待取和取衣；
4. 会员办卡、充值、折扣、次卡、积分和会员生命周期管理；
5. 短信、微信回执、催取、生日及到期提醒；
6. 小票、水洗唛、不干胶标签等多种打印；
7. 营业统计、库存催取、欠款管理、日结/月结和交班结账；
8. 员工权限、门店资料、价格字典、卡类规则及数据管理。

系统最核心的产品价值可以概括为：

> 以“衣物”为最小业务对象，以“票单”为交易载体，以“顾客/会员”为经营对象，把前台收衣、后场流转、门店收款、顾客通知和经营结账串成一个闭环。

需要特别说明：截图显示微信公众平台、线上系统、快递账户和摄像头在当前门店环境中存在“未对接、未开通或未启用”的状态。因此，本报告把这些写为“系统提供的可配置能力”，不代表该门店当时已经实际启用。

## 三、核心业务功能详解

### 1. 门店首页与前台工作台

首页承担的是“门店值班总台”角色，而不是单纯菜单页。截图可见：

- 顶部一级入口：系统设置、业务管理、日常业务、业务统计、财务管理、会员中心、微信中心；
- 快捷取衣：输入票单号后执行一键取衣；
- 信息发送：可在短信和微信之间选择，输入手机号与消息内容后发送；
- 预置消息：完工提醒、到店邀请、生日祝福、逾期催取、换季营销等；
- 顾客快速检索：支持姓名、电话、卡号；
- 顾客摘要：会员卡类、卡号、卡内余额、消费总额、首次/上次消费、有效期、地址；
- 快捷动作：收衣、取衣、办理新卡、充值、清空；
- 当日看板：今日收衣清单、今日取衣订单、件数和金额等。

这一设计说明系统的高频使用方式是：员工登录后长期停留在首页，通过快捷入口完成顾客接待，而不是在多层菜单里反复切换。

### 2. 收衣开单

收衣页是整个系统的核心页面，至少包含以下操作区。

#### 2.1 顾客识别

- 客户姓名；
- 客户电话；
- 会员卡号；
- 查找已有顾客或新增顾客；
- 办理新卡；
- 查看客户照片；
- 查询该顾客待取衣物。

#### 2.2 服务和衣物品类

截图明确显示的服务大类有：

- 干洗；
- 手洗；
- 普洗；
- 床品；
- 洗鞋；
- 洗包；
- 皮护；
- 保养；
- 染色；
- 单烫；
- 修补。

衣物可以继续按上装、下装、棉衣、裙子、大衣、布鞋、皮鞋、皮质、箱包、家居、其他等分类，也支持自定义品类、助记码检索、默认价和 B 价。

#### 2.3 单件衣物描述

每件衣物可以记录：

- 服务方式；
- 衣物名称；
- 颜色；
- 品牌；
- 备注；
- 附加项目；
- 条码；
- 附件；
- 款式；
- 分区；
- 数量；
- 原价与折价；
- 加急；
- 现场服务；
- 上门收送员。

这表明系统不是只按“一个订单一个金额”记账，而是按订单下的多件衣物逐件建档和计价。

#### 2.4 质量与证据留存

- 支持拍照；
- 支持记录污渍、磨损、起毛、少扣等衣况备注；
- 支持附件信息；
- 可打印水洗唛和附件标签；
- 可为单件衣物生成条码。

截图中实际点击拍照时提示“未启用摄像头，无法拍照”，说明页面具备能力，但当前设备没有启用。

#### 2.5 订单金额与结算

订单汇总区可见：

- 总件数；
- 原价总计；
- 折价总计；
- 附加费；
- 加急费；
- 运费；
- 核价总额。

还可以决定是否发送短信/微信回执、打印取衣联/存根、标记需要回送，以及为整单增加备注。

#### 2.6 暂存订单

系统支持把正在录入但尚未完成的收衣单“暂存挂单”，稍后调单继续收衣。这适用于顾客临时补充衣物、柜台被其他顾客打断或暂时无法确认价格的场景。

### 3. 生产流转与衣物状态

日常业务菜单明确包含：

- 收衣；
- 送洗；
- 上挂；
- 取衣；
- 重新打印；
- 票据撤销；
- 支出/收入；
- 职员留言；
- 临单录入；
- 快递收送。

系统体现出的标准流程是：

```text
收衣开单 → 送洗/生产 → 完工上挂 → 待取 → 顾客取衣或安排回送
```

#### 3.1 送洗

菜单和流程统计都能确认“送洗”是独立状态。价格资料还可为衣物设置“送洗至工厂”或“送洗至本店”，说明系统考虑了门店自产和外送工厂两种生产模式。

#### 3.2 上挂

上挂页用于把生产完成的衣物放到指定挂点或取衣输送机位置，支持：

- 查看待上挂衣物；
- 按票单号或条码查询；
- 单件选取或批量选取；
- 记录分区和挂点；
- 语音播报；
- 上挂时打印不干胶标签；
- 查看前台、生产中、完工待取件数。

#### 3.3 取衣

取衣页支持：

- 通过姓名、电话、票单号、条码号和挂点等信息查找；
- 展示订单下的逐件衣物、金额、付款方式、收衣时间和衣物状态；
- 整单取衣；
- 部分取衣；
- 标记整单未取或单件未取；
- 返工；
- 处理欠款和逾期保管费；
- 回送打印、回送通知、催取通知；
- 修改或清空挂点；
- 打印不干胶标签；
- 只查看网约订单。

取衣不是一个简单的“完成”按钮，而是同时处理衣物交付、未取留存、欠款收取、返工和回送。

#### 3.4 重打与撤销

- 可按票单号或条码号精确查找历史订单；
- 可重打取衣单、存根联、不干胶、水洗唛和附件；
- 水洗唛支持整单或单件打印；
- 票据撤销需要填写撤销原因；
- 可选择是否发送短信或微信通知。

### 4. 顾客与会员管理

系统同时维护“散客”和“会员”，并允许将散客资料合并到会员资料。

#### 4.1 散客管理

散客列表可记录和查询：

- 姓名、电话；
- 性别；
- 车牌、地址、证件；
- 顾客标识；
- 历史消费金额；
- 历史件数；
- 消费次数；
- 上次消费日期。

支持新增、修改、删除、查找和维护多个地址。

#### 4.2 会员档案

会员中心显示：

- 卡名称、卡号、姓名、电话；
- 余额、赠额、积分；
- 鞋次卡、衣次卡；
- 生日、照片、性别和标识；
- 办卡日期、有效期和上次光临；
- 历史消费、历史件数、消费次数；
- 黑名单状态。

#### 4.3 办卡

办卡页可录入：

- 姓名、电话、证件、车牌；
- 昵称、标签、生日、地址、性别；
- 顾客照片；
- 卡类型、卡号、有效期；
- 储值金额与赠送金额；
- 鞋次卡、衣次卡或月月赠；
- 是否设置密码、是否短信验证；
- 团体卡；
- 各服务类别的折扣；
- 是否打印取衣票单、是否需要回执、是否拒收营销短信、是否免收保管费。

#### 4.4 卡类规则

卡类并非只有一个统一折扣。截图显示每种卡可配置：

- 卡名称、工本费、有效期；
- 一次性赠送或按月赠送；
- 开卡和续卡最低储值；
- 是否允许前台办理、是否允许充值；
- 余额不足时是否继续享受折扣；
- 先扣本金还是赠额；
- 干洗、手洗、普洗、床品、洗鞋、洗包、皮护等分类折扣；
- 附加服务是否参与折扣；
- 次卡充值选项。

#### 4.5 会员生命周期操作

截图明确出现：

- 发行新卡；
- 老卡充值；
- 老卡升级；
- 信息修改；
- 信息打印；
- 挂失补卡；
- 过期延期；
- 积分兑换；
- 会员合并；
- 会员退卡；
- 次卡管理；
- 赠额月返；
- 会员统计与历史消费查询。

#### 4.6 积分、邀请和推广

积分规则可按会员/散客的消费金额累计，并可设置积分抵扣洗衣费。系统还提供邀请码、推荐办卡奖励、被推荐人持续消费返积分等机制。

微信推广设置进一步提供普通推广员/职业推广员、二级或三级分销、金币奖励和金币自动兑换余额。该能力在截图中属于配置功能，不表示当前门店已实际使用。

### 8. 库存、待取和催取

系统所说的“库存”主要指已经收进门店但尚未取走的衣物，而不是洗涤耗材库存。

库存待取页支持：

- 按收衣日期或单号区间排查；
- 按状态筛选已收衣、已送洗、已上挂；
- 查看衣物编号、服务、名称、颜色、条码、顾客、金额和时间；
- 常规排查、深度排查和条件查询；
- 按订单或客户查找；
- 查看需要回送、输送线、挂点使用情况；

## 四、五条关键业务闭环

### 闭环 A：普通散客收衣

```text
手机号/姓名查找散客
  → 选择洗护服务与衣物品类
  → 逐件记录颜色、品牌、瑕疵、附件、条码
  → 计算原价、折扣、附加费、加急费和运费
  → 选择付款方式
  → 打印小票/标签并发送回执
  → 进入生产流程
```

### 闭环 B：会员消费

```text
读取会员卡/手机号查找
  → 显示余额、赠额、折扣、积分和次卡
  → 按会员规则自动计价
  → 余额/赠额/次卡/现金等方式结算
  → 更新积分和消费历史
  → 打印或发送会员回执
```

### 闭环 C：衣物生产与交付

```text
已收衣
  → 送洗到本店或工厂
  → 完工上挂并登记挂点
  → 系统通知顾客可取
  → 按票单/条码/挂点查找
  → 整单或部分取衣
  → 必要时补款、收保管费、返工或安排回送
```

### 闭环 D：长期未取处理

```text
库存待取筛选
  → 按订单或顾客汇总
  → 短信/微信批量催取
  → 逾期自动计收保管费
  → 取衣时收款并完成交付
```

### 闭环 E：门店结账

```text
当班期间收衣、取衣、充值、支出和补收欠款
  → 按支付渠道汇总
  → 核对备用金与钱箱现金
  → 职员交班结账
  → 打印/发送业绩报告
  → 留存日结、月结和历史详单
```

## 六、截图中仍无法确认的事项

以下内容不能仅凭本批截图下结论：

1. 数据备份是否支持自动备份、恢复校验和备份轮换；

7. 库存盘点是衣物库存、耗材库存，还是两者都支持；
8. 数据导出格式、权限和脱敏方式；

## 八、最终判断

这套系统的核心不是某一个“功能按钮”，而是以下四类数据互相联动：

- 顾客/会员；
- 票单与付款；
- 单件衣物及其生产状态；
- 门店收入、员工和通知记录。

其中最值得保留和抽象的产品主线是：

> 快速收衣开单 → 单件衣物可追踪 → 完工上挂可定位 → 顾客通知可触达 → 取衣收款可闭环 → 当班账目可核对。

会员营销、微信推广、复杂卡类、硬件配置和高级数据工具都很丰富，但它们应当围绕这条主线服务，不能替代收衣、生产、取衣和结账这四个最核心的门店动作。
