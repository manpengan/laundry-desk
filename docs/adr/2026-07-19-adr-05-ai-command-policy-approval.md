# ADR-05: AI 命令总线 / 风险策略 / 确认与审批

- 日期：2026-07-19　状态：**Accepted**（终审裁决单独签署，2026-07-19 生效）　父文档：[总 RFC](2026-07-19-v2-productization-and-ai.md)
- 详设：架构 §6.5、§9.4–§9.6、§9.8

## 决策

1. **统一 Command/Query Bus 是唯一写入口**（V2-M1 基座）：人工按钮、AI 工具、自动化策略、Edge 离线回放走同一命令信封（含 `actor.via`、幂等键、dry_run、confirm_ref）与同一校验链（Zod → RBAC → 租户范围 → Policy/Risk → 不变量 → 事务 → 审计 → 事件）。
2. **AI Tool Registry = 命令/查询注册表的投影**，无第二套工具实现；投影附 LLM 描述与脱敏规则。
3. **单一 AI Operator Runtime**：六个场景只是预设（prompt + 工具白名单 + 风险上限 + 模型引用），不做六套 agent。
4. **风险分级 R0–R5**：R0/R1 自动（R1 附来源）；R2 敏感读取（权限+脱敏+限量）；R3 单次确认卡；R4 确认卡+二次验证/老板审批（审批中心）；**R5（权限/密钥/备份恢复/审计删除/系统设置）不进入投影，AI 不可执行**。风险随参数升级（数量/金额阈值触发 R3→R4，阈值只可调严）。
5. **删除"AI 写操作免确认"全局开关**。自动化只能通过 `automation_policies`：仅 ≤R3 工具、对象过滤器、每日次数/金额/时间窗限额、有效期、owner 审批生效、一键暂停、逐次落审计计入限额；**硬编码红线**（退款/免单/余额/权限/密钥/备份恢复/审计删除永不可自动执行）。
6. **确认卡 = WYSIWYS 服务端授权对象**（`ai_pending_actions`）：绑定 tool + 完整 args + args_hash + 操作者 + 租户 + 实体版本快照 + 有效期（默认 5 分钟）+ nonce + 幂等键；执行时重校验 hash/版本/过期/操作者/RBAC，任何参数变更即作废重出；UI 必须完整呈现被授权内容。
7. AI 安全定义：AI 可执行**当前操作员有权执行的任何已注册业务命令**，但无裸 SQL、文件系统、任意 URL、硬件原始接口、密钥、系统管理接口。
8. Prompt injection 纵深防御：最小权限 → 不可信输入隔离 → 输出 Zod+不变量验证 → 人类闸门 → 异常监控 → **红队用例集进 CI 门禁**；PII 脱敏默认开。
9. **审计同事务（二轮补丁）**：业务变更与审计写入同一数据库事务，审计失败业务回滚；应用角色对 `audit_log` 仅授 INSERT（防篡改靠数据库权限）；审计表启用 RLS，审计访问本身留痕。
10. **canonical args 服务端冻结**：确认卡创建时参数落库为唯一权威副本；确认/复核只提交 nonce，执行读取冻结参数——客户端与 AI 都不能在确认后重新提交参数。
11. **R4 两级形态**：M1 起提供**同步 step-up 复核**（具备审批权限者现场 PIN/扫码，绑定 args hash/实体版本/有效期/审批人/幂等键，原子单次消费；操作者不能复核自己）；M5 增加异步审批中心（代办/催办/看板）。`approval_request` 同样绑定上述要素且单次消费。
12. 命令元数据补齐四字段：`offline_allowed / data_classification / max_batch / result_redaction`；AI 数据表（对话/pending/审批 payload）纳入 RLS + 保留期限（默认 180 天）+ 顾客删除联动。

## 理由

AI 后置到 M5 会让 M1–M4 积累"页面能用、AI 不能安全用"的接口（二审 P0）；全局免确认是系统里最危险的权限位；确认卡若不绑定参数哈希与实体版本，存在"确认后换参数"与 TOCTOU 两类绕过（OWASP 交易授权"所见即所签"原则）。

## 否决的备选

- agent 直调 service 层（draft1 方案：绕过统一校验链，否决）。
- 全局免确认开关（draft1 方案，二审否决）。
- 引入重型 CQRS/工作流框架（总线核心是注册表+校验链，自研百行级足够）。

## 后果

- Policy Engine 与总线属核心基座，单一所有者开发，M1 交付确认卡 + step-up 闭环（R3 确认卡 M3 首用，R4 step-up 使 M2 起的退款/撤销/批量命令不必等 M5）。
- 审批中心 M5 只补**异步**形态（单级），不做多级流程引擎。
